# memo-continue

## 目标
帮助用户定位并继续编辑 `.memo/` 目录下已有的记录文件。

## 工作流

1. **扫描 `.memo/` 目录**：
   - 使用 `ls -lt` 或等效方式列出 `.memo/*.md` 文件，按修改时间排序。
2. **判断用户意图**：
   - **情况 A**：用户明确指定了文件名（如“继续编辑 `char-balance.md`”）→ 直接读取该文件。
   - **情况 B**：用户未指定文件名，但输入中包含“最新”或“latest”→ 直接使用最近修改的文件。
   - **情况 C**：用户未指定文件名，也未说“最新”→ 找到最近修改的文件，先向用户汇报文件名和修改时间，**询问**：“是否继续编辑 `char-balance.md`？（修改时间：2025-06-07）”
3. **读取文件**：使用 Read 工具读取文件内容，展示给用户。
4. **等待用户指令**：用户确认后，接收用户的修改内容，使用 Edit 或 Write 工具更新文件。

## 询问模板（情况 C）

```
最近修改的记录是：
- 文件名：char-balance.md
- 修改时间：2025-06-07

是否继续编辑这个文件？（回复“是”或“最新”即可直接打开；也可以告诉我你想编辑哪个文件）
```

## 示例

### 示例 1：用户未指定文件
用户：`继续编辑`
→ 系统列出 `.memo/` 目录，发现最近修改的是 `char-balance.md`
→ 回复用户：
```
最近修改的记录是：
- 文件名：char-balance.md
- 修改时间：2025-06-07

是否继续编辑这个文件？（回复“是”或“最新”即可直接打开；也可以告诉我你想编辑哪个文件）
```

### 示例 2：用户明确说“最新”
用户：`继续最新的记录`
→ 直接打开最近修改的文件，展示内容并等待用户编辑。

### 示例 3：用户指定文件
用户：`继续编辑 ui-layout.md`
→ 直接打开 `.memo/ui-layout.md`，展示内容并等待用户编辑。

## 注意事项
- `.memo/` 目录为空时，提示用户没有现有记录，建议先使用 `memo-write` 创建。
- 读取文件内容后，如果内容很长，可以分段展示。
- 用户回复确认后，使用 Edit 工具追加或修改内容，或询问用户具体修改位置。
